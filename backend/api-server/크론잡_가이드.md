# 정책 자동 업데이트 크론잡 가이드

## 목표
EC2에서 매일 자동으로:
1. 정책 데이터 수집 및 DB 저장
2. 블로그 글 생성
3. 썸네일 이미지 생성

---

## 빠른 시작 (EC2)

### 1. 프로젝트 준비
```bash
cd ~/capstone25-t3-kch-6/backend/api-server

# .env 파일 확인
cat .env
```

필수 환경 변수:
- `DB_URL`: PostgreSQL 연결 정보
- `YOUTHCENTER_API_KEY`: 청년센터 API 키
- `OPENAI_API_KEY`: OpenAI API 키
- `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `S3_BUCKET`: S3 설정 (선택)

### 2. 크론잡 설치
```bash
cd ~/capstone25-t3-kch-6/backend/api-server
chmod +x setup_cron.sh
./setup_cron.sh
```

실행 시간 선택 (권장: 매일 오전 3시)

---

## 시스템 흐름

```
[크론잡 실행]
    ↓
정책 수집 (update_policies.py)
    ↓
블로그 생성 (generate_all_blogs.py)
    ↓
썸네일 생성 (thumbnail_refresh.py)
    ↓
완료
```

---

## 수동 실행

### 전체 프로세스 실행
```bash
~/run_policy_update.sh
```

### 개별 실행
```bash
cd ~/capstone25-t3-kch-6/backend/api-server

# 1. 정책 수집
python3 jobs/update_policies.py

# 2. 블로그 생성
python3 jobs/blog/generate_all_blogs.py

# 3. 썸네일 생성
python3 jobs/thumbnail_refresh.py
```

### 테스트 모드
```bash
# 정책 수집 테스트 (1페이지만)
python3 jobs/update_policies.py --max-pages 1

# 블로그 생성 테스트 (5개만)
python3 jobs/blog/generate_all_blogs.py --limit 5

# 썸네일 생성 테스트 (DB 업데이트 안 함)
python3 jobs/thumbnail_refresh.py --dry-run
```

## 확인 및 관리

### 크론잡 확인
```bash
crontab -l
```

### 로그 확인
```bash
# 실시간 로그
tail -f ~/capstone25-t3-kch-6/backend/api-server/logs/cron_output.log

# 상태 로그
tail -f ~/capstone25-t3-kch-6/backend/api-server/logs/cron_status.log
```

### DB 확인
```bash
# 정책 개수
psql $DB_URL -c "SELECT COUNT(*) FROM policy_clean;"

# 블로그 개수
psql $DB_URL -c "SELECT COUNT(*) FROM blog_posts;"

# 썸네일이 있는 블로그 개수
psql $DB_URL -c "SELECT COUNT(*) FROM blog_posts WHERE thumbnail_url IS NOT NULL;"

# 최근 생성된 블로그
psql $DB_URL -c "SELECT plcy_no, blog_title, thumbnail_url FROM blog_posts ORDER BY updated_at DESC LIMIT 5;"
```
