# 정책 자동 업데이트 크론잡 가이드

## 목표
EC2에서 매일 자동으로 정책 수집 → DB 저장 → 썸네일 생성 → 블로그 생성

---

## 파일 구조

```
backend/api-server/
├── jobs/update_policies.py    # 메인 크론잡 스크립트
├── setup_cron.sh              # 자동 설정 (이것만 실행!)
└── test_update.sh             # 테스트용
```

---

## 빠른 시작 (EC2)

### 1. 코드 가져오기
```bash
ssh -i "your-key.pem" ec2-user@your-ec2-ip
cd ~/capstone25-t3-kch-2
git checkout feature/cron
git pull
```

### 2. 기본 패키지 설치 (Amazon Linux)
```bash
# Python & PostgreSQL
sudo yum install -y python3 python3-pip postgresql postgresql-devel git gcc

# 프로젝트 패키지
cd backend/api-server
pip3 install --user -r requirements.txt

# Node.js & PM2
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install --lts
npm install -g pm2
```

### 3. .env 파일 설정
```bash
cd ~/capstone25-t3-kch-2/backend/api-server
nano .env

### 4. PM2로 FastAPI 서버 24시간 실행
```bash
cd ~/capstone25-t3-kch-2/backend/api-server
pm2 start "python3 -m uvicorn main:app --host 0.0.0.0 --port 8000" --name api-server
pm2 startup
pm2 save

# 확인
curl http://localhost:8000/api/health
```

### 5. 크론잡 자동 설정
```bash
cd ~/capstone25-t3-kch-2/backend/api-server
chmod +x setup_cron.sh
./setup_cron.sh
```

→ 실행 시간 선택 (권장: 매일 오전 3시)

### 6. 테스트
```bash
# Dry-run 테스트
./test_update.sh

# 실제 실행 (1페이지만)
python3 jobs/update_policies.py --max-pages 1
```

---

## 수동 실행

```bash
# Dry-run (저장 안 함)
python3 jobs/update_policies.py --dry-run --max-pages 1

# 실제 실행
python3 jobs/update_policies.py

# 크론 헬퍼 스크립트
~/run_policy_update.sh
```

## 확인 및 관리

### 크론잡 확인
```bash
crontab -l
```

### 로그 확인
```bash
# 실시간
tail -f ~/capstone25-t3-kch-2/backend/api-server/logs/cron_output.log

# 오늘 날짜
tail -f ~/capstone25-t3-kch-2/backend/api-server/logs/update_policies_$(date +%Y%m%d).log
```

### PM2 관리
```bash
pm2 list
pm2 logs api-server
pm2 restart api-server
```

### DB 확인
```bash
psql $DB_URL -c "SELECT COUNT(*) FROM policy_clean;"
psql $DB_URL -c "SELECT plcy_no, title, created_at FROM policy_clean ORDER BY created_at DESC LIMIT 5;"
```

## 트러블슈팅

### 크론이 안 돌아가요
```bash
sudo systemctl status crond
sudo systemctl restart crond
~/run_policy_update.sh  # 수동 실행 테스트
```

### Python 모듈을 찾을 수 없음
```bash
pip3 install --user -r requirements.txt
which python3
```

### API 호출 실패
```bash
curl http://localhost:8000/api/health
pm2 logs api-server
pm2 restart api-server
```

### DB 연결 실패
```bash
sudo systemctl status postgresql
psql $DB_URL -c "SELECT 1;"
```

## 시스템 흐름

```
[크론잡 매일 오전 3시]
    ↓
[update_policies.py]
    ↓
[youthcenter API 호출] → 최신 정책 목록
    ↓
[DB 조회] → 기존 정책 ID
    ↓
[신규 정책 필터링]
    ↓
[DB 저장] → policy_raw, policy_clean
    ↓
[썸네일 생성] → FastAPI /api/thumbnails/auto → S3
    ↓
[블로그 생성] → blog_posts 테이블
    ↓
[로그 기록] → logs/
```
