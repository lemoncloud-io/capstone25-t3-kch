# 썸네일 생성 문제 해결 가이드

## 문제 상황
- 크론잡이 실행되어 블로그 글은 생성됨 ✅
- 하지만 썸네일 이미지는 생성되지 않음 ❌

## 원인
기존 크론잡 스크립트에서 썸네일 생성 단계가 누락되어 있었습니다.

### 기존 흐름
```
정책 업데이트 → 블로그 생성 → ❌ (썸네일 누락)
```

### 수정 후 흐름
```
정책 업데이트 → 블로그 생성 → 썸네일 생성 ✅
```

---

## EC2에서 수정하는 방법

### 방법 1: 자동 수정 스크립트 사용 (권장)

```bash
# 1. EC2 접속
ssh -i "your-key.pem" ec2-user@your-ec2-ip

# 2. 프로젝트 디렉토리로 이동
cd ~/capstone25-t3-kch-6/backend/api-server

# 3. 최신 코드 가져오기
git pull

# 4. 수정 스크립트 실행
chmod +x fix_cron_thumbnail.sh
./fix_cron_thumbnail.sh
```

이 스크립트가 자동으로:
- 기존 크론잡 헬퍼 스크립트 백업
- 썸네일 생성 로직 추가
- 옵션: 기존 블로그 글에 대한 썸네일 즉시 생성

---

### 방법 2: 수동 수정

기존 크론잡 헬퍼 스크립트(`~/run_policy_update.sh`)를 직접 편집합니다.

```bash
nano ~/run_policy_update.sh
```

**추가할 코드 위치:**
블로그 생성 완료 후, 다음 코드를 추가합니다:

```bash
# 블로그 생성이 성공했으면 썸네일도 생성
if [ $BLOG_EXIT_CODE -eq 0 ]; then
    echo "[INFO] 썸네일 생성 시작..." >> $LOG_FILE
    $PYTHON_BIN $PROJECT_DIR/jobs/thumbnail_refresh.py >> $LOG_FILE 2>&1
    THUMBNAIL_EXIT_CODE=$?
    echo "[INFO] 썸네일 생성 완료 (Exit code: $THUMBNAIL_EXIT_CODE)" >> $LOG_FILE
    EXIT_CODE=$THUMBNAIL_EXIT_CODE
else
    echo "[ERROR] 블로그 생성 실패. 썸네일 생성을 건너뜁니다." >> $LOG_FILE
    EXIT_CODE=$BLOG_EXIT_CODE
fi
```

저장 후 종료 (Ctrl + X → Y → Enter)

## 기존 블로그 글에 썸네일 생성하기

이미 생성된 블로그 글들에 대해 썸네일을 생성하려면:

```bash
cd ~/capstone25-t3-kch-6/backend/api-server

# 테스트 (실제 DB 업데이트 안 함)
python3 jobs/thumbnail_refresh.py --dry-run

# 실제 실행
python3 jobs/thumbnail_refresh.py
```

이 스크립트는:
- `thumbnail_url`이 비어있는 블로그 글을 찾음
- 각 블로그에 맞는 썸네일 자동 생성
- S3 또는 로컬에 저장
- DB 업데이트


브라우저에서 블로그 목록 페이지를 새로고침하여 썸네일이 표시되는지 확인합니다.

## 추가 정보

### 썸네일 생성 프로세스
1. DB에서 `thumbnail_url`이 비어있는 블로그 조회
2. 각 블로그의 정책 정보 조회
3. LLM을 사용하여 썸네일 문구 생성 (또는 폴백)
4. 카테고리에 맞는 배경 이미지 선택
5. PIL로 텍스트 렌더링 및 이미지 생성
6. S3 업로드 또는 로컬 저장
7. DB에 `thumbnail_url` 업데이트

### 파일 위치
- 썸네일 생성 스크립트: `backend/api-server/jobs/thumbnail_refresh.py`
- 썸네일 생성 로직: `backend/api-server/routes/thumbnails.py`
- 배경 이미지: `backend/api-server/assets/backgrounds/`
- 폰트: `backend/api-server/assets/fonts/`
